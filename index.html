Flappy Bird

by: Ian Smee

This game was made for my level 2 project at The League of Amazing Programmers.
Click here to download and play!

1. Why I Made This Game

I chose this game because it was very simple and a good first game to make, and because I could add onto the game to make it more interesting.

2. What I Learned

I learned that my enemy model was not obviously an enemy.

3. Architecture of My Game

My game consists of 9 classes. Several of them are objects, and the others control, manipulate, or help manage the objects. The object manager class controls all the objects and creates them inside of itself each time the game is run. The other classes also help setup the whole frame for the game, with the different screens and more.

4. Technical Implementation

I am proud of the gravity aspect in the game, as it is quite realistic. I had to ask for help to create it but tweaked it to make it more accurate. It makes it so that jumping resets the downward momentum of the bird that is being controlled and makes it positive, but constantly decreasing, which creates a nice jumping motion.

5. Most Challenging Part

Setting up the frame for the project as a whole was the most difficult part of the game. All of the technical aspects that needed to be made before other things could took a lot of time to set up. Eventually I finished all of them and then I could work on the easier parts of the project, like making objects and manipulating them.

6. Future Fixes and Updates

I could make it so that after a certain amount of time or levels, the whole map would change and more obstacles would appear, over and over again until the player beat the game, maybe with a boss fight at the end.
